# Sistema de reservas para negocios locales

Aplicación web de reservas de citas pensada para negocios de servicios (peluquerías, clínicas, estética, consultas). Permite a un cliente reservar sin necesidad de registrarse: elige servicio, profesional, día y hora disponible, y recibe la confirmación al instante.

> 🔗 Demo en vivo: https://reservas-demo.romerogill.com/

---

## Características

- **Reserva sin registro.** El cliente reserva en cuatro pasos con solo su nombre y email.
- **Cálculo de disponibilidad en tiempo real.** El sistema genera los huecos libres cruzando el horario del profesional, las reservas ya existentes y los bloqueos (vacaciones, festivos), respetando la duración de cada servicio.
- **Multi-negocio.** La base de datos está diseñada para alojar varios negocios, cada uno con sus servicios, profesionales y horarios.
- **Protección frente a reservas duplicadas.** Antes de confirmar, se vuelve a verificar que el hueco sigue libre para evitar que dos clientes reserven la misma hora.
- **Diseño responsive y accesible.** Interfaz adaptada a móvil y a pantallas grandes.

---

## Arquitectura

El proyecto separa la lógica de negocio de la interfaz:

- **`src/lib/disponibilidad.js`** — Motor de cálculo de huecos. Función pura, sin dependencias de React ni de la base de datos, lo que la hace fácil de probar de forma aislada. Es el núcleo técnico del proyecto.
- **`src/lib/reservasService.js`** — Capa de datos: consulta Supabase (horarios, reservas, bloqueos) y delega el cálculo en el motor de disponibilidad.
- **`src/lib/supabase.js`** — Cliente de conexión a Supabase.
- **`src/PaginaReserva.jsx`** — Interfaz pública de reserva.

El motor de disponibilidad está cubierto con tests unitarios (Vitest) que verifican los casos clave: generación de huecos, exclusión de horas ocupadas, respeto de bloqueos, días sin horario y descarte de horas pasadas.

## Puesta en marcha en local

Requisitos: Node.js 18 o superior y una cuenta de Supabase.

```bash
# 1. Clonar el repositorio
git clone https://github.com/AromeroGill/reservas-app.git
cd reservas-app

# 2. Instalar dependencias
npm install

# 3. Configurar las variables de entorno
#    Crea un archivo .env en la raíz con tus credenciales de Supabase:
#    VITE_SUPABASE_URL=https://tu-proyecto.supabase.co
#    VITE_SUPABASE_ANON_KEY=tu-anon-key

# 4. Arrancar en modo desarrollo
npm run dev

# 5. (Opcional) Ejecutar los tests
npm test
```

El esquema de la base de datos (tablas de negocios, servicios, profesionales, horarios, bloqueos y reservas) se crea desde el editor SQL de Supabase.

# Booking system for local businesses

Appointment booking web app aimed at service businesses (hair salons, clinics, beauty studios, private practices). Customers can book without signing up: they pick a service, a professional, a day and an available time slot, and get instant confirmation.

> 🔗 Live demo: https://reservas-demo.romerogill.com/
---

## Features

- **No sign-up required.** Customers book in four steps with just their name and email.
- **Real-time availability.** Free slots are generated by combining the professional's working hours, existing bookings and blocked periods (holidays, time off), while respecting each service's duration.
- **Multi-business ready.** The database is designed to host several businesses, each with its own services, staff and schedules.
- **Double-booking protection.** Slot availability is re-checked right before confirming, preventing two customers from booking the same time.
- **Responsive and accessible.** Interface adapted for both mobile and large screens.

---

## Tech stack

| Layer | Technology |
|-------|------------|
| Frontend | React + Vite |
| Database & API | Supabase (PostgreSQL) |
| Data security | Row Level Security (RLS) |
| Deployment | Cloudflare Pages |

---

## Architecture

Business logic is kept separate from the UI:

- **`src/lib/disponibilidad.js`** — Availability engine. A pure function with no React or database dependencies, which makes it easy to test in isolation. This is the technical core of the project.
- **`src/lib/reservasService.js`** — Data layer: queries Supabase (schedules, bookings, blocks) and delegates the calculation to the availability engine.
- **`src/lib/supabase.js`** — Supabase connection client.
- **`src/PaginaReserva.jsx`** — Public booking interface.

The availability engine is covered by unit tests (Vitest) verifying the key cases: slot generation, exclusion of busy hours, respect for blocked periods, days with no schedule, and discarding past times.

## Getting started locally

Requirements: Node.js 18+ and a Supabase account.

```bash

git clone https://github.com/AromeroGill/reservas-app.git
cd reservas-app

# 2. Install dependencies
npm install

# 3. Set up environment variables
#    Create a .env file in the root with your Supabase credentials:
#    VITE_SUPABASE_URL=https://your-project.supabase.co
#    VITE_SUPABASE_ANON_KEY=your-anon-key

npm run dev

# 5. (Optional) Run the tests
npm test
```

The database schema (tables for businesses, services, staff, schedules, blocks and bookings) is created from the Supabase SQL editor.
